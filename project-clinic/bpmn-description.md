## **Схема: Обработка заявки нового клиента**

### Основная информация:
**Название:** Обработка заявки нового клиента 

**Уровень сложности:** Средний+ 

**Инструмент:** StormBPMN

### Цель процесса:
Регистрация и согласование нового клиента с проверкой данных в двух системах и автоматическим согласованием при просрочке.

### Участники:
- **Клиент**
- **Менеджер**
- **Система А** (основная система)
- **Система Б** (система проверки)
- **Руководитель**

### Ключевые элементы:
- Boundary Timer Event (2 дня на согласование)
- Автоматическое согласование по таймеру
- Проверка данных в Системе Б
- XOR Gateways для развилки по результатам проверки
- Обновление статуса и уведомления

### Особенности:
- **Автосогласование:** если руководитель не рассмотрел заявку за 2 дня → система согласует автоматически
- Две системы взаимодействуют между собой
- Уведомления клиента при отказе и согласовании
- Уведомление менеджера о результате
- Слияние веток (ручное и автоматическое согласование)
