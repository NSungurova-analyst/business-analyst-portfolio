## ** Описание к BPMN - схеме: Заказ пиццы**

### Основная информация:
**Название:** Процесс заказа пиццы с доставкой 
**Уровень сложности:** Средний 
**Инструмент:** StormBPMN

### Цель процесса:
Обработка заказа пиццы от клиента через мобильное приложение с доставкой и сбором обратной связи.

### Участники:
- **Клиент**
- **Оператор/Администратор**
- **Система**
- **Курьер**
- **Кухня/Повар**
- **Сотрудник отдела качества**

### Ключевые элементы:
- Проверка наличия ингредиентов
- KDS (Kitchen Display System) вместо бумажных стикеров
- Parallel Gateway для параллельной работы кухни и курьера
- Boundary Timer Event (2 дня для сбора обратной связи)
- Link Event для оптимизации схемы
- Разные способы оплаты (онлайн/при получении)

### Особенности:
- **Современный подход:** KDS вместо бумажных стикеров
- Изменение состава заказа при отсутствии ингредиентов
- Параллельное приготовление пиццы и поиск курьера
- Уведомление администратора через Telegram
- **Сбор обратной связи** через 2 дня после доставки
- Link Event для избежания длинных линий на схеме
- Два сценария завершения маршрута курьера (доставка/отмена)
