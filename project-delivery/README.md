# 🍔 Проект: Сервис доставки еды

## 📌 Проблема
65% клиентов испытывают тревогу после оформления заказа: "Где мой курьер?", "Когда привезут?". Это приводит к 30% повторных звонков в поддержку. Курьеры жалуются на низкий заработок из-за неоптимальной маршрутизации, текучка достигает 40% в месяц.

## 💡 Решение
Спроектировала систему отслеживания заказов и оптимального распределения заказов для курьеров:
- Отслеживание курьера на карте в реальном времени
- Push-уведомления о смене статуса заказа
- Автоматическая группировка заказов по географическому признаку
- Построение оптимального маршрута для курьера
- Система бонусов за множественные заказы

## 📦 Артефакты

### User Stories
[user-stories.md](./user-stories.md) — 2 истории с контекстом и критериями приёмки:
- US-1: Отслеживание заказа в реальном времени (карта, ETA, уведомления)
- US-2: Оптимальное распределение заказов для курьера (группировка, маршрутизация, бонусы)

### BPMN-схема
[bpmn-refund.png](./bpmn-refund.png)
[Подробное описание процесса →](./bpmn-description.md)

**Описание:**
- BPMN-схема процесса обработки возврата денег (фокус на exception handling).
- Основные этапы: подача заявки → проверка менеджером → обработка финансистами → возврат средств.
- Роли: Клиент, Система, Менеджер, Финансовый отдел, Руководитель.

**Почему этот процесс:**
В отличие от интернет-магазина (где показан основной процесс заказа), здесь я сфокусировалась на обработке исключений — это показывает понимание полного цикла работы сервиса и умение проектировать не только happy path.
