## **Схема: Согласование нового поставщика**

### Основная информация:
- **Название:** Процесс согласования нового поставщика 
- **Уровень сложности:** Высокий 
- **Инструмент:** StormBPMN

### Цель процесса:
Согласование нового поставщика с участием нескольких директоров, автоматическими напоминаниями и возможностью доработки данных.

### Участники:
- **КМ** (Контрактный менеджер)
- **МОЗ** (Менеджер по оформлению закупок)
- **Система** (автоматические процессы)
- **НОЗ** (Начальник отдела закупок)
- **Директора** (по закупкам и по продажам)

### Ключевые элементы:
- Параллельное согласование двумя директорами (Parallel Gateway)
- Boundary Timer Events (3 часа на подтверждение)
- Автоматические напоминания при просрочке
- Цикл доработки данных (МОЗ ↔ НОЗ)
- Message Flow между процессами (передача в "Оформление тестового заказа")

### Особенности:
- При отсутствии ответа от директора в течение 3 часов → автоматическое напоминание
- Возможность возврата на доработку с циклической проверкой
- Интеграция с соседним процессом через Message Events
